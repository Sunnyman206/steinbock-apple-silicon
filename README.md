# Steinbock on Apple Silicon (M1/M2/M4) — Native ARM64 Setup

## Overview

This guide documents how to run the full Steinbock IMC preprocessing and segmentation pipeline natively on Apple Silicon Macs (M1, M2, M4). The official Steinbock Docker image (`ghcr.io/bodenmillergroup/steinbock`) is built for `linux/amd64` and uses TensorFlow compiled with AVX instructions that Rosetta cannot emulate, causing DeepCell/Mesmer segmentation to fail. This guide builds a native ARM64 image from source to work around that limitation.

---

## Requirements

- macOS on Apple Silicon (M1, M2, or M4)
- Docker Desktop with Rosetta emulation enabled (Settings → General → "Use Rosetta for x86/amd64 emulation")
- Git
- ~20 minutes for the initial build
- ~5 GB disk space for the Docker image

---

## Quick Start

### Step 1 — Build the native ARM64 image

```bash
bash install_steinbock_arm64.sh
```

This will clone the Steinbock repo, build a native ARM64 Docker image called `steinbock-arm64`, and add a convenience alias to `~/.zshrc`.

### Step 2 — Open a new terminal (to load the alias)

```bash
steinbock --version
```

### Step 3 — Run the pipeline (see full protocol below)

---

## Full IMC Pipeline

### Directory structure

Steinbock expects this layout:

```
project_folder/
  raw/          ← place your .mcd files here
  panel.csv     ← generated automatically, then edited
  img/          ← generated by steinbock
  masks/        ← generated by steinbock
  intensities/  ← generated by steinbock
```

### 1. Extract images from raw MCD files

```bash
cd /path/to/project_folder

steinbock preprocess imc panel

steinbock preprocess imc images --hpf 50
```

`--hpf 50` applies a hot pixel filter (recommended for IMC data).

### 2. Edit panel.csv for segmentation

Open `panel.csv` and set the `deepcell` column:

- **1** = nuclear channel (e.g., Ir191 or Ir193)
- **2** = cytoplasmic/membrane channel (e.g., CD45, CD31)
- **leave empty** = not used for segmentation (do NOT use `0`)

> **Critical:** Empty cells become NaN and are skipped. If you fill unused rows with `0`, steinbock will count it as a third channel group and segmentation will fail with: `Invalid number of aggregated channels: expected 2, got 3`.

Example rows in panel.csv:
```
channel,name,keep,ilastik,deepcell,cellpose
Ir191,191Ir,1,48,1,
Dy162,CD45,1,32,2,
Nd142,CD20,1,13,,
```

### 3. Run DeepCell segmentation

```bash
steinbock segment deepcell --minmax
```

Successful output will look like:
```
INFO steinbock - masks/your_image_001.tiff
INFO steinbock - masks/your_image_002.tiff
```

### 4. Measure single-cell intensities

```bash
steinbock measure intensities --img /data/img --masks /data/masks -o /data/intensities
```

### 5. Measure region properties (morphology)

```bash
steinbock measure regionprops --img /data/img --masks /data/masks -o /data/regionprops
```

### 6. Export to CSV or AnnData

```bash
steinbock export csv --intensities /data/intensities --regionprops /data/regionprops -o /data/cells.csv

steinbock export anndata --intensities /data/intensities --regionprops /data/regionprops -o /data/cells.h5ad
```

## Known Limitations on Apple Silicon

- **Ilastik** — bundled as an x86 binary; may have reduced performance under Rosetta
- **CellProfiler** — included in the image but GPU acceleration not available
- **Large images** — memory-intensive segmentation may require reducing Docker Desktop memory allocation (Settings → Resources)

## Troubleshooting

### `Invalid number of aggregated channels: expected 2, got 3`

The `deepcell` column in `panel.csv` has `0` values in non-segmentation rows. Replace all `0` values with empty cells:

```bash
python3 -c "
lines = []
with open('panel.csv') as f:
    for line in f:
        parts = line.strip().split(',')
        if len(parts) >= 5 and parts[4] == '0':
            parts[4] = ''
        lines.append(','.join(parts) + '\n')
with open('panel.csv', 'w') as f:
    f.writelines(lines)
"
```

### `Error: No such command 'bash'`

The steinbock container does not expose a shell via the default entrypoint. Run commands directly:

```bash
steinbock segment deepcell --help
```

### `no matching manifest for linux/arm64/v8`

You are using the official pre-built image, not the ARM64 build. Run `install_steinbock_arm64.sh` and use the `steinbock-arm64` image.

### AVX / TensorFlow error under Rosetta

```
The TensorFlow library was compiled to use AVX instructions, but these aren't available on your machine.
```

This confirms you are running the x86 image through Rosetta. Use the ARM64 image built by `install_steinbock_arm64.sh`.

## Visualization with Napari

```bash
conda activate imc-napari   # or your napari environment

python3 -c "
import napari, tifffile, numpy as np

img  = tifffile.imread('img/your_image_001.tiff')
mask = tifffile.imread('masks/your_image_001.tiff')

print('Cells segmented:', len(np.unique(mask)) - 1)

viewer = napari.Viewer()
viewer.add_image(img[47], name='Ir191_DNA',  colormap='gray')
viewer.add_image(img[31], name='CD45',       colormap='green', blending='additive')
viewer.add_labels(mask,   name='masks')
napari.run()
"
```

Adjust channel indices to match your panel (0-indexed).
